## Recent Releases

Stay informed about Embarque updates. You receive notifications via email and the dashboard when new versions deploy. Each release focuses on stability, CFDI compliance, and streamlined embarque workflows.

<Callout kind="info">
  Check your dashboard at https://embarque.mx for upgrade status. Most updates apply automatically with zero downtime.
</Callout>

<Update label="2024-10-15" description="v1.3.0" tags={["feature", "improvement"]}>

## New Features

- Added Carta Porte 3.1 bulk validation for faster embarque processing
- Integrated SAT real-time facturación status checks directly in the dashboard

## Improvements

- Optimized cobranza reports to load `<2s` even with `>10,000` records
- Enhanced mobile view for on-the-road embarque confirmations

## Bug Fixes

- Fixed duplicate facturas generation during high-volume peaks
- Resolved PDF export issues with accented characters in client names

</Update>

<Update label="2024-09-25" description="v1.2.1" tags={["bugfix", "security"]}>

## Bug Fixes

- Patched SAT API token refresh to prevent 401 errors after idle sessions
- Corrected embarque tracking sync delays with third-party carriers

## Security Updates

- Updated encryption for cuentas por cobrar sensitive data
- Added rate limiting to prevent abuse on facturación endpoints

</Update>

<Update label="2024-09-10" description="v1.2.0" tags={["feature", "breaking"]}>

## New Features

- Introduced automated cobranza reminders via WhatsApp integration
- New analytics dashboard for embarque velocity and SAT rejection rates

## Breaking Changes

- Updated facturación schema to fully comply with CFDI 4.0 – review your templates
- Deprecated legacy Excel import; migrate to CSV format

## Improvements

- Reduced facturación generation time by `40%` with parallel processing

</Update>

## Known Issues & Workarounds

<ExpandableGroup>
  <Expandable title="High-volume SAT rejections during peak hours" default-open="true">
    Issue affects `>500` facturas/hour. Workaround: Batch submissions in groups of 100. Fix in v1.3.1.
  </Expandable>
  <Expandable title="Custom PDF templates not rendering logos">
    Ensure logos are `<500KB` PNG files. Update templates via dashboard settings.
  </Expandable>
</ExpandableGroup>
